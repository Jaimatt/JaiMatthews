The Password company is a small project made for a Year 11 computer science class. In it, users can make an account, sign in, sign out, and do other things commonly associated with an account system. Yes, it's a website that is solely an account system. I had some interesting school projects.

The Account Creation Screen

Being predominantly a learning opportunity, the website has a number of... uh.. quirks. First of all, it runs entirely in vanilla JavaScript. Of course, this means you can just open the inspect element, but it doesn't strictly matter as the point of the project was never achieving genuine security.

Another "quirk" is the extremely strict requirements for making a password. This was because a part of the project was to practice this kind of error/ exception handling.

Stringent Password Requirements

Being relatively new to JavaScript (and programming as a whole) at the time of writing the code for this project, I was not super familiar with the different variable formats and had to figure out the best way of saving users' usernames and passwords to the system.
